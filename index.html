QuartoPad · private beta

Quarto authoring, rendering, chunks, data, and citations in one surface.

QuartoPad opens from R, runs locally on your machine, and keeps the document, preview, datasets, and references close enough to work as one loop instead of five tabs.

Editor loop

Write, render, and inspect without leaving the document.

Edit `.qmd`, render HTML or slides, and keep the output visible in the same browser window instead of bouncing between tools.

Persistent R session

Chunk execution behaves like a real working notebook.

Inline execution sends code into a live R session so state carries between chunks instead of resetting on every action.

Reference workflow

Data previews and citations stay close to the draft.

Browse datasets, inspect `.bib` files, and search references beside the document rather than in separate helper apps.

TrimWrite · internal beta

A draft analyzer for vague phrasing, AI habits, and over-polished prose.

The public repo already contains a substantial analyzer UI. The right public message, though, is still beta: useful enough to test, not yet positioned as fully launched.

How the products fit together

One brand. Two precise tools. One writing loop.

01

TrimWrite is the umbrella name.

Use the brand to hold the point of view: local-first tools for people who write with data, code, references, and revision.

02

QuartoPad stays product-specific.

The name is sharp and credible for the Quarto and R audience. It should not be flattened into a generic brand label.

03

TrimWrite the app stays distinct too.

That keeps the analysis product clear: it is not another editor, it is the second tool in the line.

Current build status

What the repo audit says right now.

The site should lead with honesty. QuartoPad is further along as a product. TrimWrite has a real working UI but is still in the hardening phase. trimwrite.com should present both as coming soon.

QuartoPad

Closer to broader testing.

The editor, rendering flow, citations, and dataset tools already justify a serious private-beta presentation.

TrimWrite

Feature-rich, but not launch copy yet.

The public repo includes a large app surface, but its own status docs still flag production-readiness work as incomplete and the repo is currently archived. That is not a good primary landing destination for users yet.

trimwrite.com

Now positioned as the umbrella site.

The public homepage should present the two products together and collect early interest instead of implying either is already open for general release.
