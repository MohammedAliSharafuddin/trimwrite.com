First Product

QuartoPad turns Quarto authoring into a working surface.

The point is not another generic writing app. The point is a tighter environment for reproducible documents where authoring, code, previews, and citations stay in the same flow.

Split editor

Source, preview, and console live together.

Open a `.qmd`, run code against a persistent R environment, and render the document without leaving the editor surface.

  • Inline chunk execution
  • Live Quarto rendering
  • HTML, slides, PDF, and Word output
Data-aware

Preview datasets without breaking writing flow.

Browse CSV or RDS files in the same workspace and keep the document grounded in the data you are actually using.

Citation workflow

Search references where you write.

Work with local `.bib` files, CrossRef search, and Zotero-friendly workflows without switching into a separate citation manager tab.

Workflow

What TrimeWrite is aiming for.

01

Open a real project directory

TrimeWrite products should start from the files you already own, not a forced cloud workspace.

02

Keep writing and computation adjacent

Code, output, datasets, and prose belong in one operational loop, especially for technical writing and research reports.

03

Ship focused tools, not a vague platform

QuartoPad stays specific. TrimeWrite becomes the product family that can later house other writing tools with equally clear scope.
